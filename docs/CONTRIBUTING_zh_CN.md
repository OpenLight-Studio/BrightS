# 为 BrightS 做贡献

感谢您考虑为 BrightS 做出贡献！我们欢迎来自社区的贡献。  
感谢你为 BrightS 贡献代码！我们欢迎来自其他组织的贡献！

## 如何贡献

1. Fork（派生）本仓库  
2. 为你的功能或错误修复创建一个新分支  
3. 进行你的修改  
4. 确保你的代码遵循项目的编码规范  
5. 如适用，请添加测试  
6. 提交 Pull Request（拉取请求）

## 编码规范

- 遵循项目中现有的代码风格  
- 编写清晰、具有描述性的注释  
- 使用有意义的变量名和函数名  
- 保持函数专注且规模合理  
- 适当地处理错误

## 报告问题

请使用 GitHub 问题跟踪器来报告错误或提出功能建议。请包含以下内容：  
- 对问题的清晰描述  
- 复现步骤（针对错误）  
- 期望行为与实际行为  
- 相关的日志或错误信息

## 开发环境设置

有关设置开发环境的说明，请参阅 [BUILD.md](docs/build.md)。

## 许可证

通过向 BrightS 贡献代码，即表示您同意您的贡献将根据本项目的许可证进行授权。
